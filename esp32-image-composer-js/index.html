Manual Flashing Instructions

Use these commands to flash your firmware manually from the command line:

Workflow Information
ESP-IDF Workflow (Recommended):
Build with ESP-IDF → generates .elf file
Use espflash: espflash app.elf
Automatically handles bootloader + partition table + app
Manual Binary Workflow:
Individual .bin files for each partition
Use esptool.py for precise control
Good for custom bootloaders or recovery
Using espflash (Recommended)
Flash ELF Application (Primary Method):
espflash --chip esp32p4 --baud 460800 --flash-size 16mb --flash-freq 80m --monitor

Usage: espflash --chip esp32p4 app.elf (flashes bootloader + partition table + app)

Requirements: ESP-IDF built ELF file (not .bin), espflash handles conversion automatically

Flash Individual Binary Partitions (Advanced):
espflash --chip esp32p4 --baud 460800 --format esp-idf

Advanced: Use only when you need specific bootloader/partition table binaries

Serial Monitor Only:
espflash --chip esp32p4 --monitor

Connects to device and opens serial monitor without flashing

Flash Downloaded Complete Image:
espflash --chip esp32p4 --baud 460800 --format esp-idf --partition-table-offset 0x0 esp32-p4-complete-firmware-.bin

For binary images downloaded from this tool. Use --format esp-idf for compatibility.

Using esptool.py
Flash Individual Partitions:
python -m esptool --chip esp32p4 -b 460800 --before default_reset --after hard_reset write_flash --flash_mode dio --flash_size detect --flash_freq 80m
Flash Complete Downloaded Image:
python -m esptool --chip esp32p4 -b 460800 --before default_reset --after hard_reset write_flash --flash_mode dio --flash_size detect --flash_freq 80m 0x0 esp32-p4-complete-firmware-.bin

Example: python -m esptool --chip esp32p4 -b 460800 --before default_reset --after hard_reset write_flash --flash_mode dio --flash_size detect --flash_freq 80m 0x0 ~/Downloads/esp32-p4-complete-firmware-1765963895195.bin

Pattern: 0x0 <path/to/filename.bin> (address + filename, not just filename)

Wokwi CLI (Simulation Validation)
Validate ESP-IDF Build:
wokwi-cli --elf build/esp32_p4_graphical_bootloader.elf

Test your firmware in web simulator before hardware flashing

Advanced Validation:
# Test with custom diagram wokwi-cli --elf build/app.elf --diagram diagram.json # Use specific Wokwi version wokwi-cli --elf build/app.elf --version latest

Advanced options for custom test scenarios and version control

Wokwi Validation Workflow
  1. Build your firmware with ESP-IDF: idf.py build
  2. Validate in simulator: wokwi-cli --elf build/app.elf
  3. Fix any issues found in simulation
  4. Then flash to hardware: espflash app.elf
esptool.py Setup
  1. Install: pip install esptool
  2. Put ESP32-P4 in download mode (BOOT + RESET)
  3. Identify serial port
  4. Run: python -m esptool --port /dev/ttyUSB0 ...
espflash Setup (Recommended)
  1. Install: cargo install espflash
  2. Or via pip: pip install espflash
  3. Auto-detects ESP32-P4 and optimal settings
  4. Usage: espflash app.elf (flashes complete firmware)
  5. Monitor: espflash --monitor (serial monitor only)
Note: espflash works with ELF files from ESP-IDF builds (.elf), not .bin files
Wokwi CLI Setup (Validation)
  1. Install: pip install wokwi-cli
  2. Test firmware in web simulator
  3. Usage: wokwi-cli --elf build/app.elf
  4. Validates before hardware flashing
  5. Support: ESP32, ESP32-S2, ESP32-S3, ESP32-C3
Note: Always validate with Wokwi before flashing to hardware!
Common Serial Ports
Linux: /dev/ttyUSB0, /dev/ttyACM0
macOS: /dev/cu.usbserial-XXXX, /dev/cu.usbmodemXXXX
Windows: COM1, COM2, COM3... (check Device Manager)
Tip: Use espflash list-ports to find devices
Common Issues & Solutions
❌ Wrong: esptool ... filename.bin
✅ Right: esptool ... 0x0 filename.bin
esptool requires address + filename pairs
❌ Wrong: espflash filename.bin
✅ Right: espflash app.elf
espflash prefers ELF files from ESP-IDF builds
